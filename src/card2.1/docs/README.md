## 🚨 重要提醒

### 开发前必读
1. **先看快速开始** - 理解基本概念和开发流程
2. **参考现有组件** - 查看 `src/card2.1/components/test/` 下的示例
3. **遵循命名规范** - 使用kebab-case命名和标准目录结构
4. **测试验证** - 在Visual Editor中验证组件功能

### 常见陷阱
- ❌ 忘记添加`dataSources`配置导致数据源插槽不显示
- ❌ 缺少`settingConfig`导致配置面板空白
- ❌ 组件分类配置错误导致组件不显示
- ❌ 属性暴露配置缺失导致交互配置失败
